assistant: **Redis it is** for the cache then. With 3 instances, in-process would give inconsistent cache hits. Use redis-py with a connection pool, set TTLs per cache key type (short for user data, longer for reference data).

Or run as `python -m pytest` which adds the current dir to sys.path automatically.

user: thanks the pytest fix worked. on caching, we cache user preferences (TTL 5min) and product catalog (TTL 1h). what key scheme?
assistant: Use namespace prefixes to avoid collisions:
- User prefs: `user:prefs:{user_id}` → TTL 300s
- Product catalog: `catalog:product:{product_id}` → TTL 3600s
- Use `user:prefs:*` for bulk invalidation on user data changes

Set `maxmemory-policy allkeys-lru` in Redis config so it gracefully evicts under memory pressure rather than returning errors.
